Solar Buddy

Current release posture

What Solar Buddy does now, what is committed next, and what remains outside the current public release promise.

Live now

  • Address-first property lookup and real map rendering
  • Browser location lookup that fills the map and address form
  • Street and terrain map modes for property inspection
  • Roof polygon drawing directly on the live map
  • Persist roof geometry with the property record
  • Feed saved roof geometry into the estimate sizing
  • Surface clearer estimate assumptions and confidence on the results
  • Model monthly production from roof-backed sizing and explicit system-loss assumptions
  • Save solar report snapshots with the current property record
  • Promote saved planning reports into a shareable homeowner quote workflow
  • Refine production inputs with inferred roof-facing tilt and nearby site-context losses
  • Use utility-aware savings so manual electricity-rate entry is no longer the only credible path
  • Run address-context quick estimates before a roof polygon is drawn, with canopy, building, and terrain context in the model
  • Infer parcel roof capacity from matched building footprints so quick estimates can size without a manual kW guess

Committed next

  • Add multi-plane roof segmentation and setback-aware roof editing so drawn roofs stop behaving like a single flat capacity bucket

Later, not committed

  • Parcel-grade roof enrichment on every property
  • Quote-grade guarantees before lead qualification
  • Premium data layers by default in the public workflow

Garden Buddy

Current release posture

Garden Buddy uses the same property foundation, but its release scope and roadmap are tracked separately from the solar workflow.

Live now

  • Address-first property lookup and real map rendering
  • Browser location lookup that fills the map and address form
  • Street and terrain map modes for lot inspection
  • Garden polygon drawing directly on the live map
  • First-pass sun classes and seasonal exposure summaries for drawn zones
  • Garden zones saved to the current property record and restored when the property is reopened
  • Recent saved garden plans can reopen directly from the shared property library
  • First-pass planting guidance tied to each saved zone's exposure profile
  • Property climate context, estimated hardiness band, average temperature and humidity, and live irradiance on the active garden plan
  • First obstruction-aware property context using nearby building footprints and terrain cues, saved with the property and rendered on the garden map

Committed next

  • Extend the saved yard context toward repeat-use planning and later automation
  • Push beyond the current building-and-terrain slice into stronger parcel or canopy-aware shade context

Later, not committed

  • Tree-perfect shade simulation in the first release
  • Sensor automation before saved property context exists
  • Premium canopy and terrain enrichment on every property
